Getting Started with PHeasel

Installing PHeasel

Download the latest release and unzip it to the htdocs directory of your web server.

FYI:

Creating your first page

Your website will be living within the site subdirectory. Let's create a simple page in there: create a new file and call it page.first.php. It is important to have page in the file name to let PHeasel know that the file contains a page. You can also name it first.page.php if you prefer.

File: page.first.php 
        <!DOCTYPE HTML>
        <html>
        <head>
            <!--
            ${${config
                url  = /first/
                id   = firstpage
                name = First Page
            }$
            -->
            <title>My first page</title>
        </head>
        <body>
            <h1>Hello, welcome to the first page</h1>
        </body>
        </html>

The page options are enclosed within ${config and }$.

  • url: the path where this page will be accessible on the webserver, e.g. on your local webserver the example page would be accessible at http://localhost/first/
  • id (optional): a unique identifier for this page, you will need it when creating links to this page, e.g. firstpage
  • name (optional): a short name for this page, can be used for link labels, espacially in generated components like sitemaps or navigation elements
More about page markup options

Linking pages

Let's create a second page, so that we have a place to link to the page above:

File: page.second.php 
    <!DOCTYPE HTML>
    <html>
    <!-- pheasel
    url  = /second/
    id   = secondpage
    name = Second Page
    /pheasel -->
    <head>
        <title>My second page</title>
    </head>
    <body>
        <h1>This is the second page</h1>
        Take me to the first:
        <a href="${url pageid=firstpage}$">${pagename}$</a>
    </body>
    </html>

Again, we have assigned url, id and name in the page configuration. We have also added a link back to our first page:

  • ${url pageid=firstpage}$ inserts the (relative) link to the page with id firstpage, in our example this is /first/.
  • ${pagename}$ will always be replaced with the name of the last page that a link has been generated for, in this case First page. Of course you can insert any other text instead, but doing it this way will make work with navigational components a lot easier.

BTW: PHeasel placeholders

As you can see in the example above, some patterns in your markup are processed by PHeasel and replaced with something else. By default, these patterns start with ${ and end with }$. If you actually need to display these character combinations in your page (e.g. when creating a PHeasel tutorial page :)), you can escape them like this: $'{ and }'$.
If these are completely in your way (maybe because another tool you use has similar patterns), you can change them in pheasel_config.php. That said, be sure to read the comments in there.

Using templates

Templates are a good way to re-use markup that wraps your page content on several pages, e.g. navigation or footer elements.

File: tmpl.main.php 
    <!DOCTYPE HTML>
    <html>
    <!-- pheasel
    id   = main
    /pheasel -->
    <head>
        <meta charset="utf-8"/>
    </head>
    <body>
    <div class="header">Navigation go here</div>
    ${page}$
    <div class="footer">Footer links go here</div>
    </body>
    </html>

A template file needs to have tmpl in the filename, we call ours tmpl.main.php.
There is not much to configure within the template:

  • id (optional): a unique identifier for this template, you will need it to assign it to your pages, e.g. main
  • ${page}$ inserts the actual page content of the requested page
File: page.first.php (excerpt) 
    [...]
    <!-- pheasel
    url      = /first/
    id       = firstpage
    name     = First Page
    template = main
    /pheasel -->
    [...]

You still need to reference the template from the page to be wrapped by it. Just add
template = main
to the configuration section of any page that should be using the template.

Noteworthy: if you have a look at the resulting page's source, you will see that it's <head> section contains the template's and the page's <head> content.
